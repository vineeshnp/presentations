Scaling

Vertical scaling and Horizontal scaling.
Socket scaling using broker.
game-state scaling using cache servers.
Message queues.
Reverse proxy using nginx.

* Vertical scaling is done by adding more resource to a single system.
* Horizontal scaling is done by distributing load into multiple workers.
* For horizontal scale, we need a centralized game-state store, viz the cache
* To enable socket connected to different servers to communicate, we need message-broker
* Message queues could be used to ensure sequential execution and delayed timers.
* For any system with multiple cores, its good to use cluster and ngnix with reverse proxy to ensure all cores are used.

Challenges

  • Node is not designed for game backend.
  • Handling computations on event loop.
  • Importance of doing CPU intensive tasks using golang/rust.
  • Need of timers.
* Game involves lot of computation and node does not do well in computational intensive tasks,
* But the latest version of node have workers to do the tasks, viz a great thing
* Rust and golang could do computations much faster, so a rust executable script
can be called as an io operation and load on event loop can be minimized.
* Real game use timers a lot, some cases its okay to store timestamp in database
other cases we might need timer servers, to trigger delayed actions.

Suggestions

  • forget es5 approach.
  • es6 / typescript will be lifesavers.
  • Security with tokens
  • rather than writing from scratch, you could use frameworks.[But don't expect a long term support]
  • do profiling to find out time consuming functions.
  • For production use game engines.
* es6 made js more of a kind of good language
* each socket connections integrity is to be validated, jwt is a good option
* Using frameworks is a great option, but LTS isn't guaranteed.
* Profiling with google chrome dev tools, let us know which function is throttling.
* For production, game engines are always advisable, because that takes care of the
plumbing work of components, we only have to worry about logical parts.
